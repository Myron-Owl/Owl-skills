# 玩家现实隧道模式

分析游戏机制、体验反馈、留存、付费、公平性或玩家冲突时必须加载本文件。目标不是替设计者或玩家辩护，而是找出“设计意图”和“实际可感知体验”之间的转换误差。

## 先补齐最小上下文

从输入中提取；缺失但不阻断时明确假设：

- 机制和核心规则
- 设计目标与期望玩家行为
- 目标玩家、游玩阶段和平台情境
- 资源来源、消耗、恢复与失败代价
- 玩家能看到的反馈和可做的选择
- 当前证据：行为数据、访谈、社区文本或团队判断

不要从一句玩家评论直接外推全体玩家。

## 感知链路

沿链路定位断点：

`客观规则 → 可见信息与反馈 → 玩家因果解释 → 情绪/价值判断 → 行为选择 → 社区叙事`

设计者通常掌握规则全貌，玩家只能根据局部反馈推断系统意图。规则数学正确不等于体验可理解，也不等于体验公平。

## 选择相关玩家隧道

不要穷举所有人群。选择最可能产生不同解释的三到六类：

- 新手与熟练玩家
- 休闲玩家、时间有限玩家与高投入玩家
- 探索者、优化者、竞争者、收藏者或叙事玩家
- 付费与非付费玩家
- 当前获益者与受损者
- 单人体验者、组队者与社区意见领袖

对每类回答：他看见什么、如何解释、在保护什么价值、会采取什么行为、什么证据能验证。

## 核心体验维度

- **能动性**：玩家是否拥有真实且可理解的选择？
- **可读性**：结果能否追溯到规则、选择和反馈？
- **公平感**：规则是否一致，代价是否与责任匹配？
- **胜任感**：失败是否提供学习线索，成长是否可感知？
- **时间尊重**：等待、重复和损失是否被视为有意义投入？
- **选择意义**：选项是否改变策略，而非伪选择或最优解陷阱？
- **收益契约**：付出、风险、奖励和惊喜是否符合玩家预期？
- **身份与社交**：机制是否威胁玩家身份、地位、归属或表达？

## 常见意图—体验错位

| 设计者模型 | 玩家可能模型 | 可观察风险 |
|---|---|---|
| 资源管理 | 自由被定时剥夺 | 到点退出、囤积、抱怨“逼下线” |
| 提高挑战 | 系统作弊或浪费时间 | 归因不清、失败后不重试 |
| 促进长期目标 | 人为拖延内容 | 日常疲劳、捷径付费争议 |
| 保持经济稳定 | 奖励被克扣 | 奖励价值感下降、套利行为上升 |
| 鼓励多样选择 | 强迫使用不喜欢的玩法 | 选择集中、反感轮换规则 |
| 增加随机惊喜 | 结果不受控制 | 连败记忆、阴谋叙事、信任下降 |
| 便捷付费 | 先制造痛点再收费 | “付费解问题”叙事扩散 |

错位只是候选模型，必须用行为和反馈验证。

## 游戏分析输出

### 设计者现实隧道
写出目标、机制因果、预期行为和成功指标。

### 玩家现实隧道
按关键分群写出可见信息、推断、情绪价值和行为。

### 认知错位
指出链路在哪一步发生歧义，以及哪些 UI、文案、节奏、奖励或规则在强化该解释。

### 竞争模型
至少比较：理解/反馈问题、激励/数值问题、核心心理需求问题。按具体设计增删，不要预设玩家“不理解”。

### 验证计划
将证据分为：

- 行为：进入率、完成率、中断点、重试、资源囤积、替代玩法、流失。
- 表达：访谈用词、客服主题、社区叙事、误解频率。
- 分群：新老、付费、投入时长、胜负状态或内容阶段的差异。
- 实验：信息增强、数值调整、主动选择、补偿方式或节奏变化的对照。

不要只看平均值；平均值容易掩盖分群相反反应。

### 优化方向
按成本和可逆性排列：

1. 先修复反馈、预告、因果可读性和选项表达。
2. 再调整摩擦、频率、阈值、奖励与失败代价。
3. 若心理契约本身冲突，重构机制而非继续包装。
4. 为每项改动写明预期改变的玩家解释和观测指标。

## 疲劳机制示例框架

- **设计者模型**：疲劳限制单日最优刷取，制造队伍轮换和长期资源规划。
- **玩家模型 A**：疲劳是有策略价值的预算；前提是消耗收益清楚、恢复可规划、替代方案有效。
- **玩家模型 B**：疲劳是系统禁止继续玩；当恢复被动、惩罚陡峭或替代玩法无价值时更可能出现。
- **玩家模型 C**：疲劳是付费压力；当痛点与付费恢复紧邻、非付费恢复不可控时更可能出现。
- **关键区分证据**：玩家是在规划前产生选择，还是在耗尽后直接退出？不同分群是否把付费恢复视为便利、必需或惩罚？
- **低成本实验**：增强消耗预告与收益展示，提供一次主动恢复选择，对比耗尽后的退出率、轮换率和负面叙事。

不要把“增加可见收益”当通用答案；若玩家反对的是自由被限制，包装收益不会修复根本契约。
