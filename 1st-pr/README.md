# 第一性原理思维 (First Principles Thinking)

> 一个 Claude Code skill — 将问题拆解至基本真理，从底层重新构建解决方案。

## 概述

这个 skill 让 Claude 使用**第一性原理思维**来分析和解决问题。它不是直接跳到"最佳实践"或"行业标准答案"，而是：

1. 拆解问题到不可再分的基本元素
2. 识别并挑战所有隐含假设
3. 从基本真理出发重新构建解决方案
4. 通过验证实验检验方案的可靠性

适用于：架构设计、复杂 Bug 排查、产品创新、战略决策、技术选型等。

## 使用方法

在 Claude Code 中，有两种方式调用：

### 方式 1：Slash Command

```
/1st-pr 为什么房价会上涨？
```

### 方式 2：自然语言触发

直接说出包含触发词的问题：

```
用第一性原理思考：为什么电动车加速更快？
```

支持的中文触发词：
- 第一性原理
- 拆解问题
- 从根本思考
- 打破思维定势
- 归零思考

### 方式 3：在对话中主动调用

当 Claude 识别到以下场景时会自动建议使用此 skill：
- 传统方法无法解决的复杂问题
- 需要质疑现有假设时
- 反复出现的 Bug 需要根因分析
- 架构设计和技术选型

## 输出模板

分析输出采用标准化结构，确保结果一致可读：

```markdown
## 第一性原理分析：[问题名称]

### 1. 问题定义
### 2. 拆解至基本元素
### 3. 假设挑战
### 4. 从零重建
### 5. 验证计划
### 6. 结论
```

## 内置框架

| 框架 | 适用场景 |
|------|---------|
| 框架 A: 软件架构拆解 | 系统设计、性能优化、技术选型 |
| 框架 B: Bug 根因拆解 | 复杂 Bug 排查、线上事故分析 |
| 框架 C: 产品/功能设计拆解 | 需求分析、功能设计、UX 改进 |

## 特色

- **认知偏差预警** — 识别确认偏误、沉没成本谬误等5种常见思维陷阱
- **领域应用框架** — 3个即插即用的分析框架覆盖主流场景
- **对话流程引导** — 定义AI与用户的分步交互流程
- **完成清单** — 确保分析完整性的6项检查标准
