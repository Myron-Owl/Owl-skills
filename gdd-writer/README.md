# 策划文档撰写技能 (GDD Writer)

> GDD = Game Design Document（游戏设计文档）。
> 一个 Claude Code skill — 严格遵循《策划文档 AI 协作规范》，按模板和规则生成 CC战棋 项目策划文档。

## 概述

这个 skill 让 Claude 能够**按照规范生成标准化的游戏策划文档**。它不是一个自由写作工具，而是一个受规则约束的写作框架：

1. **写作前准备** — 确认文档类型、编写 ai_context、读取依赖文档、补充提问
2. **模板化生成** — 5 种文档类型各有固定模板（system / world / narrative / data / guide）
3. **规范检查** — 写作后按 16 项质量清单自动检查
4. **AI 行为约束** — 明确禁止空泛内容、未确认信息、格式违规等

## 使用方法

在 Claude Code 中调用：

```
/gdd-writer 帮我写一个 X 系统的策划文档
/gdd-writer 按照规范生成命中系统策划文档
/gdd-writer 检查这篇文档是否符合规范
```

自然语言也可以触发：
- "写一个势头系统的策划文档，参考现有命中系统文档"
- "帮我更新队伍管理界面的策划文档到 v0.2"

## 为什么需要这个技能

CC战棋 项目有一套详细的[《策划文档 AI 协作规范》](../策划文档规范.md)，包含：

| 类别 | 内容 |
|------|------|
| 文档结构规则 | YAML 头部、标题层级、顺序约束 |
| 信息组织规则 | 表格使用场景、代码块规范、概述写法 |
| 5 种文档模板 | system / world / narrative / data / guide |
| AI 约束 | 22 条必须遵守、25 条禁止行为 |
| 版本管理 | 版本号规则、更新记录保护、文档状态流转 |

直接记忆和执行所有规则对于普通对话来说太复杂。这个 skill 将规范内化为可执行的框架，确保每次生成的文档都自动符合规范。

## 文档类型

| 类型 | 用途 | 模板章节 |
|------|------|---------|
| `system` | 系统设计 | 概述→核心公式→数值表→规则明细→功能流程→边界条件 |
| `world` | 世界观设定 | 概述→核心数据→关键设定→内部关联 |
| `narrative` | 叙事文案 | 概述→剧情线→关键角色 |
| `data` | 数值数据 | 概述→数值总表→数值来源说明 |
| `guide` | 规范指南 | —（按需组织） |

## 核心原则

1. **有效信息优先** — 只写会影响决策、实现、验收或理解的内容
2. **AI 写内容，人做决策** — AI 完成初稿和一致性检查，策划做内容决策
3. **格式一致** — 文件命名、YAML 头部、标题编号、代码块标记全局统一
4. **可追溯** — 更新记录、版本号、跨文档引用完整可查
5. **先确认后写入** — 未确认的关键信息不写进文档
