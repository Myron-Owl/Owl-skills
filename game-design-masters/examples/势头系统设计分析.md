# Momentum 势头系统设计分析

**输入：**
```
Analyze: 战棋RPG势头系统
  - 团队共用运势值（-6~+10），自然趋中7天/点
  - 燃烧势头（需≥+3）→ 悔棋 + 命中率×1.5~2.0
  - 负势头区域目前无机械惩罚（仅UI颜色变化）
  - 积累途径：8种（探索/首杀/完成任务/连胜等），大部分被动触发
Mode: breakdown + critique
Depth: high
Lens: auto
```

**核心问题识别：**
1. **P0：** 负数区域无实质内容（-1和-6感受无区别）
2. **P0：** 燃烧成本固定，高势头缺乏差异化回报（+3和+10燃烧的成本/回报比不平滑）
3. **P1：** 积累路径太被动，战斗中玩家几乎无法主动获取势头

**改进建议：**
- 负数区域补充自动Miss惩罚（D100个位数 ≤ |势头| → 触发）
- 高势头燃烧提供额外效果层（致命一击、无视减伤等）
- 增加保留势头的被动收益（如+6以上全队+5%命中率）
